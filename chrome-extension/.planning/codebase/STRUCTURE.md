## Directory Purposes

**src/**
- Purpose: All TypeScript/React application source code
- Contains: Entry points (background, content scripts, UI components), shared libraries
- Key files: `background.ts` (main orchestrator), `content-*.ts` (platform-specific), `lib/*` (business logic)

**src/lib/**
- Purpose: Shared business logic and utilities
- Contains: Storage abstractions, messaging protocol, CSV parsing, template resolution
- Modules:
  - `storage.ts` — TypeScript types and chrome.storage wrappers
  - `messaging.ts` — `sendToBackground()` sender with type safety
  - `csv-parser.ts` — CSV parsing, template/spin resolution, jitter, sleep utility

**src/popup/**
- Purpose: Quick-access popup (extension icon click)
- Contains: React component showing site status, daily quota, links to Gmail/WhatsApp
- Dependency on lib: `messaging.ts` (fetch daily count)

**src/panel/**
- Purpose: Full-featured sidebar panel for job control
- Contains: CSV upload, template editor, settings overrides, progress tracking, logs
- Dependency on lib: `messaging.ts`, `csv-parser.ts`, `storage.ts` (types)

**src/options/**
- Purpose: Settings persistence page
- Contains: Form for all `ExtensionSettings` with validation
- Dependency on lib: `messaging.ts`, `storage.ts` (types)

**public/**
- Purpose: Static HTML and icon assets
- Contains: `popup.html`, `panel.html`, `options.html` (entry points for React), icon PNGs
- Build: Copied to `dist/` by vite.config.ts post-build plugin

**tests/e2e/**
- Purpose: End-to-end testing via Playwright
- Contains: Fixtures (custom Playwright extension-specific setup), test specs, mock pages
- Key files:
  - `fixtures.ts` — `context` and `extensionId` fixtures, `setStorage()`, `clearStorage()`, `FAST_SETTINGS`
  - `global-setup.ts` — Pre-test build hook
  - `01-*.spec.ts` through `05-*.spec.ts` — Test scenarios (Gmail send, WhatsApp send, daily limit, settings, CSV parsing)

**tests/fixtures/**
- Purpose: Test data (CSV files)
- Contains: Sample contact CSVs (email and WhatsApp variants)

**dist/**
- Purpose: Built extension output
- Generated by Vite from src/ + public/
- Structure: `background.js`, `content-gmail.js`, `content-whatsapp.js`, `popup.js`, `panel.js`, `options.js`, HTML files, manifest, icons

**Testing:**

## Naming Conventions

**Files:**

- Kebab-case for multi-word files: `content-gmail.ts`, `bulk-sender-toggle` (not BulkSenderToggle.ts)
- Index pattern for React entry points: `index.tsx` (bootstrap) + `Component.tsx` (implementation)
- Descriptive names for test specs: `01-gmail-bulk-send.spec.ts`, `02-whatsapp-bulk-send.spec.ts`
- Lib utilities: Single-concept names: `storage.ts`, `messaging.ts`, `csv-parser.ts`

**Directories:**

- Lowercase, plural for collections: `src/lib/`, `src/popup/`, `src/panel/`, `src/options/`
- Lowercase, singular for feature domains: `tests/e2e/`, `public/icons/`

**TypeScript/React:**

- Interfaces: PascalCase, prefixed with descriptive noun: `ExtensionSettings`, `Job`, `Contact`, `WaJobState`, `MessageAction`
- Types: Union types for enums: `type SendingStatus = 'idle' | 'sending' | 'cooldown' | 'completed'`
- Functions: camelCase, verb-first: `resolveTemplate()`, `resolveSpin()`, `sendToBackground()`, `parseCSV()`, `loadContactsFromStorage()`
- Constants: SCREAMING_SNAKE_CASE for module-level: `SETTINGS_KEY`, `MAX_CONTACTS_WARNING`, `TTL_MS`
- React Components: PascalCase: `App`, `Popup`, `Options`
- Event handlers: `handle[Event]`: `handleFileUpload()`, `handlePanelMessage()`, `handleDrop()`

## Where to Add New Code

**New Feature (e.g., new message template system):**
- Core logic: `src/lib/csv-parser.ts` (extend `resolveTemplate()`)
- UI integration: `src/panel/App.tsx` (template editor section)
- Persistence: `src/lib/storage.ts` (add to `ExtensionSettings` interface if needed)
- Testing: `tests/e2e/05-*.spec.ts` (add test case)

**New Component/Module (e.g., batch scheduler):**
- Implementation: `src/lib/scheduler.ts` (new file)
- Integration: `src/background.ts` (call scheduler in message handler)
- Tests: `tests/e2e/new-scheduler.spec.ts`

**Utilities (e.g., sanitizer, validator):**
- Shared helpers: `src/lib/utils.ts` or domain-specific (`src/lib/validation.ts`)
- Export from lib module
- Used by content scripts and UI

**Content Script Platform (e.g., new platform like Telegram):**
- Template: Copy `src/content-gmail.ts` → `src/content-telegram.ts`
- Register in `manifest.json` content_scripts array
- Add entry point in `vite.config.ts` rollupOptions.input
- Add toggle in panel mode switcher
- Create E2E test in `tests/e2e/06-telegram-bulk-send.spec.ts`
